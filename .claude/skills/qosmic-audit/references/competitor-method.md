# Competitor analysis — method

Goal: 3–4 real competitors, found dynamically, never hardcoded. Must generalize
to a store you've never seen.

## Steps
1. **Classify the store** from crawl artifacts: read the homepage + PDP text and
   `meta` to extract category, product types, price band, and the core
   jobs-to-be-done (e.g. "ginger candy + functional nausea relief", "loose-leaf
   wellness teas + mugs"). Note the brand's distinctive edge (specialization,
   proof, content moat, formats).
2. **Find competitors via web search.** Run 2–3 queries like:
   - `"<primary category> brands"` / `"best <category> for <use-case>"`
   - `"<store name> alternatives"` / `"<store name> competitors"`
   - A use-case query for a high-intent job (`"natural motion sickness remedy brands"`).
   Use the WebSearch tool. Prefer direct-to-consumer storefronts and category
   leaders the merchant would actually be compared against. Mix 1 mass-market
   incumbent + 2–3 closer DTC peers when possible.
3. **For each competitor**, capture: domain, one-line positioning, and **what
   they make easier** for the shopper (the single thing their site does better —
   symptom-led nav, flavor sampler, retailer handoff, bundle builder, quiz).
   Optionally fetch the competitor homepage to ground this.
4. **Name the audited store's edge** per row — what it has that they don't.
5. **"Pattern to adapt"** must map to a concrete opportunity on the audited store
   (ideally one of your 10 experiments). This is what makes the table actionable.

## Quality bar
- Competitors must be plausible and real (cite the domain). No fabricated brands.
- Each row earns its place: it teaches one transferable move.
- Don't pad to 4 if only 3 are genuinely comparable.

## Generalization note
Because competitors come from category-classification + live search, the same
method works for a tea store, a skincare brand, or a hardware shop. Nothing
here is store-specific.
